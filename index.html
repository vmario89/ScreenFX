ScreenFX

A plugin library for JavaFX 8 adding features to resize and arrange stage windows for multiple screens

Download this project as a .zip file Download this project as a tar.gz file

Welcome to ScreenFX!

What is ScreenFX?

ScreenFX is a library for JavaFX 8 which allows you to install a small and lightweight popup container to your application stages. This popup has abilities to handle positioning and resizing events of each stage. Please read further to learn how to use.

Editor's note

This tool is inspired originally by the tool "Asus Multiframe" which was only capable to handle two screens. As the tool multiframe does not work an java applications and because of the screen number limitation i decided to write my own tool box.

Project requirements

This library is written for JavaFX 8 and was built with JDK 1.8.0-b121. If you compile the sources with JDK 1.7 it could mabye work 100% without any trouble.

How it works?

As you can see the layout reacts on the total count of screens. If only one screen is available, ScreenFX will only show one (the default colored main screen #1). If more screens are installed all secondary screens get a darker color than screen #1.

How to use?

The installation of ScreenFX is very easy. You do not need to initialize the tool manually. You only have to install the tool once per stage. Therefore the only requirement is that the stage contains a total minimum of one node. Tie the screenfx-current.jar into your JavaFX project and call the function like this:

Button button = new Button("Show ScreenFX");
ScreenFX screenFX = new ScreenFX();
screenFX.install(button);

If you want more access to ScreenFX you can adjust some properties (as listed in the features list):

Button button = new Button("Show ScreenFX");
Properties sfxProperties = ScreenFX.getScreenFXProperties();
sfxProperties.put("UIConventionOverride", new Boolean(false));
sfxProperties.put("allowTooltips", new Boolean(true));
sfxProperties.put("exitDelayTime", new Long(200l));
sfxProperties.put("popupKeyCodeCombination", 
    new KeyCodeCombination(KeyCode.D, KeyCombination.SHIFT_DOWN));
screenFX.setScreenFXProperties(sfxProperties);
screenFX.install(button);

ScreenFX can be used with normal builder-like source code as well as using with FXML and a controller class. You can directly install ScreenFX into the initialize() method of your controller. See the example distributed with the library.

Features

General

  • compatible up to 9 screens (cubic layout, master screen in middle of cube)
  • standard language support german + english
  • should be compatible with all operating systems (tested only Windows 7/8/8.1)

Tooltips

Functions

  • includes calculations for maximum real screen size or the screen size respecting elements like windows taskbar which reduce the area for resizing windows
  • global configuration options which are:
    • change key combination
    • enable/disable tooltips
    • enable/disable overriding of ui conventions (if enabled you can resize the window even it is in fullscreen mode)
    • add custom icon set
    • add custom language resource
    • change the delay time to exit ScreenFX when moving mouse out
  • configuration is global so you can access one static class to controll all the ScreenFX containers
  • for each stage you can allow an own ScreenFX container separately
  • resize window to screen halfs or quarters
  • put window to screen edges or corners
  • set to fullscreen (fully fullscreen or fullscreen showing taskbar)
  • maximize window in width, height or both
  • jump the window to each screen you want
  • access ScreenFX through key combination (default: ctrl + s) or via button pressing (you need a JavaFX component which extends the class ButtonBase)
